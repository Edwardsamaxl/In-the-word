# 《字里行间》设计文档

> 基于《字里行间》产品需求文档 V1.2  
> 文档版本：V1.0  
> 目标平台：抖音互动空间黑客松现场、移动端扫码体验、桌面浏览器体验  
> 设计基准：412 × 732，9:16 固定竖屏画布  
> 制作约束：26 小时，总包体不超过 8MB

## 0. 文档定位

本文档将 PRD 中的产品规则转译为可执行的视觉、排版、交互和动效规范，供 UI、动效和前端实现共同使用。

规范状态分为三类：

- **已锁定**：来自 PRD V1.2，实施时不得自行修改。
- **设计补充**：在不改变玩法的前提下补全视觉和交互细节。
- **待确认**：PRD 内存在冲突或第二关尚未定稿，需要团队在开发前决定。

## 1. 设计方向

### 1.1 方向锁定

- **用户与场景**：游园会现场的新玩家，单次体验约 2 分 30 秒；旁观者需要在两米外看懂关键演出。
- **审美方向**：数字纸页、宋体铅字、温润墨色、克制留白。不是复古书卷，也不是传统游戏 HUD。
- **核心记忆**：玩家把“明月”撞成真正的月亮，月亮穿过纸页跟随玩家进入《故乡》，最终照亮变蓝的海。
- **硬约束**：固定字符网格、固定竖屏画布、无自动换行、键盘与触屏意图等价、总包体不超过 8MB。
- **标志性交互**：文字的字面意义改变排版和物理世界，交互反馈必须同时影响字符、路径和页面。

### 1.2 同类产品提炼

| 参考 | 可借鉴原则 | 本项目中的转译 |
|---|---|---|
| 微信读书 | 熟悉的纸张比例、排版和留白降低理解成本 | 首屏伪装为正在写下的阅读页，不出现游戏菜单 |
| Device 6 | 文字排版本身构成空间和导航 | 换行、回读、下沉和越界就是关卡路线 |
| Baba Is You | 文字直接决定规则，而非解释规则 | “光”“霜”“举”“低”“海”分别产生对应物理效果 |

### 1.3 三行设计纲领

- **视觉命题**：一张安静的暖白数字纸页，墨色文字保持克制，只有规则被触发时排版才出现戏剧性变化。
- **内容路径**：诗被写下 → 方块诞生 → 光与霜 → 举头撞月 → 低头下沉 → 月落《故乡》 → 回读入海 → 海水变蓝 → 结算分享。
- **交互命题**：平时运动直接、无惯性；关键机关采用减速、整行破碎、纸页褪色和颜色渗透，让每次异常都有明确因果。

### 1.4 品牌关键词

**文学、克制、异变**

视觉判断顺序：

1. 第一眼必须像阅读页。
2. 第二眼才发现方块可以移动。
3. 关键时刻整张纸页违背阅读规则。
4. 蓝色只属于终章，不得提前透支。

## 2. 视觉主题与氛围

### 2.1 基础氛围

页面是一张没有明显边框的数字纸页。背景带轻微暖度，文字接近墨黑，整体低饱和。普通状态不使用装饰纹理、卡片、渐变标题、发光描边或传统游戏控件。

桌面端以同一竖屏画布居中显示，画布外侧使用更暗一档的暖灰留白。外侧区域不可承载玩法信息，避免形成“网页包着手机”的产品感。

### 2.2 视觉节奏

| 阶段 | 视觉密度 | 页面温度 | 动态强度 |
|---|---:|---:|---:|
| 开场书写 | 低 | 暖白 | 低 |
| 正常行走 | 中 | 暖白 | 低 |
| 光 | 中 | 暖黄 | 中 |
| 霜 | 中 | 冷白 | 中 |
| 子弹时间 | 中 | 冷灰 | 中高 |
| 撞月 | 低转高 | 墨黑与留白 | 极高 |
| 下沉 | 低 | 褪白 | 高 |
| 《故乡》探索 | 高 | 暖白逐步偏青 | 中 |
| 海水终章 | 中转低 | 海蓝 | 极高 |
| 结算 | 低 | 深海蓝 | 低 |

### 2.3 视觉锚点

唯一贯穿两关的视觉锚点是月亮。

- 第一关生成时：32px 实心墨色圆形，边缘略有墨迹扩散。
- 穿页时：保持实体，不溶解为字符。
- 第二关落定时：由墨色缓慢过渡为低饱和暖金。
- 终章时：月亮保持暖金，海面转蓝，形成全片唯一的暖冷对比。

## 3. 色彩系统

PRD 中的 Hex 色值是实现真源。CSS 中可同时维护 OKLCH 变量，但不得因转换改变肉眼效果。

| Token | 色值 | 用途 |
|---|---|---|
| `--paper` | `#F7F6F2` | 主纸张背景 |
| `--paper-warm` | `#F9F4E8` | “光”触发后的纸张暖色 |
| `--paper-cool` | `#F0EFE9` | 子弹时间背景 |
| `--ink` | `#20201E` | 正文、玩家、主要信息 |
| `--ink-muted` | `#8A8984` | 作者、提示、页码、重置 |
| `--moon-light` | `#E8D9A8` | 月光和第二关暖金月亮 |
| `--sea-mist` | `#9EC4D6` | 海水渗透中段 |
| `--sea` | `#176B9B` | 终章海蓝 |
| `--sea-deep` | `#074A73` | 最终深海和玩家终态 |
| `--letterbox` | `#E9E7E0` | 桌面画布外侧区域 |
| `--frost` | `#E9EDF0` | 霜花和裂纹高光 |

### 3.1 色彩使用规则

1. 蓝色在终章前不得作为按钮、提示、链接或状态色出现。
2. 暖金只用于月光、月亮和月亮倒影，不用于普通强调。
3. 关键词触发前保持与正文相同的 `--ink`，不能预先高亮。
4. 彩色背景上的文字不得使用中性灰，结算页文字使用海蓝同色相的浅色阶。
5. “碧绿”机关若保留，只允许出现一次低饱和淡绿涟漪，持续不超过 1.2 秒，避免破坏蓝色终章的奖励感。

## 4. 字体与排版

### 4.1 字体选择

**主字体：思源宋体 / Source Han Serif SC**

选择理由：它具有明确的中文印刷气质，字面稳定，适合固定字符网格，也能在正文、提示和结算之间保持同一世界观。

字体加载顺序：

```css
font-family: "Source Han Serif SC", "Noto Serif CJK SC", "Songti SC", serif;
```

字体只保留项目实际使用汉字、数字、拉丁字符和标点，目标子集不超过 600KB。

### 4.2 字体层级

| 层级 | 字号 | 字重 | 行高 | 字距 | 用途 |
|---|---:|---:|---:|---:|---|
| 正文字符 | 22px | 500 | 46px | 0 | 固定网格中的文章 |
| 结算主句 | 24px | 500 | 36px | -0.012em | “你真的游到了海水变蓝。” |
| 作者 | 18px | 400 | 28px | 0.08em | 李白、鲁迅 |
| 机关提示 | 14px | 300 | 22px | 0.02em | 文学化提示 |
| 结算数据 | 16px | 400 | 26px | 0.04em | 用时、步数 |
| 按钮文字 | 15px | 500 | 20px | 0.04em | 截图分享、再走一次 |
| 分享话题 | 14px | 400 | 22px | 0.06em | `#一直游` |

用时和步数必须启用等宽数字：

```css
font-variant-numeric: tabular-nums;
```

### 4.3 字符网格

- 第一关固定为 2 行 × 12 格。
- 每格逻辑宽度 22px，行高 46px。
- 标点独占一格。
- 行文本使用 `white-space: nowrap`，严禁自动换行。
- 每个字符节点必须保留 `data-row`、`data-col`、`data-char`。
- 字符视觉中心与碰撞中心必须使用同一套网格计算。
- 正文整体宽度为 264px，水平居中于 412px 画布。

### 4.4 排版禁区

- 不使用粗黑体做标题。
- 不使用手写字体模拟文学感。
- 不使用竖排正文。
- 不使用文字阴影或描边。
- 不通过增大字距制造“高级感”。
- 不在机关字下方加下划线、圆点或色块提示。

## 5. 布局原则

### 5.1 固定画布

```text
设计画布：412 × 732
安全区：左右 28px，上 28px，下 24px
正文区：宽 264px，水平居中
顶部信息区：高 48px
主要舞台区：顶部约 150px 至底部约 620px
```

桌面端缩放规则：

- 画布按比例整体缩放，不重新排版。
- 最大高度不超过视口高度。
- 不允许单独改变字号、行高或字符格。
- 画布两侧使用 `--letterbox`。

移动端缩放规则：

- 以 `min(100vw / 412, 100vh / 732)` 计算统一缩放。
- 保持 412 × 732 内部坐标系不变。
- 触屏命中区域以未缩放逻辑坐标计算后映射。
- 处理 `env(safe-area-inset-top)` 和 `env(safe-area-inset-bottom)`，但不得挤压内部画布。

### 5.2 页面层级

从底到顶：

1. `paper-layer`：纸张背景及整体色温变化。
2. `text-layer`：作者、正文字符、提示和结算文字。
3. `actor-layer`：玩家方块、拖尾、标点回退。
4. `world-layer`：月亮、海平面、月亮倒影。
5. `fx-canvas`：墨迹粒子、霜花、碎片、颜色渗透。
6. `control-layer`：重置按钮，始终可操作。

除重置按钮外，不增加常驻 UI。

### 5.3 间距系统

使用 4px 基础单位：

```text
4 / 8 / 12 / 16 / 24 / 32 / 48 / 64
```

正文内部严格服从字符网格，不使用该间距系统调整单字位置。

## 6. 组件与状态

### 6.1 玩家方块

- 尺寸：15 × 15px。
- 默认填充：`--ink`。
- 中心留 2 × 2px `--paper` 小点。
- 默认拖尾：最长 18px，持续 180ms。
- 跳跃拖尾：最长 34px，持续 350ms。
- 海中终态：颜色过渡到 `--sea-deep`。
- 玩家不是圆角角色，不添加表情、眼睛、描边或阴影。

### 6.2 重置按钮

- 图标：14px 单色循环箭头。
- 视觉颜色：`--ink-muted`。
- 实际命中区：40 × 40px。
- 默认无底色、无边框。
- 悬停：图标 opacity 从 0.65 到 1。
- 按下：`scale(0.94)`，持续 100ms。
- 焦点：2px 墨灰焦点环，外偏移 3px。
- 必须提供 `aria-label="重新开始当前关卡"`。

### 6.3 文学提示

- 位置：画面上方中部或右上安全区，根据演出避让正文。
- 字号：14px，Light。
- 颜色：`--ink-muted`。
- 最大宽度：220px。
- 进入：800ms opacity 0 到 1，translateY 4px 到 0。
- 停留：2 秒，或直到玩家执行正确动作。
- 退出：800ms opacity 1 到 0。
- 同一时刻只允许一条提示。
- 提示不得放进气泡、卡片或带箭头浮层。

### 6.4 结算按钮

结算页是全流程唯一允许出现成组按钮的页面。

**截图分享**

- 背景：月亮暖金 `--moon-light`。
- 文字：`--sea-deep`。
- 高度：44px。
- 圆角：4px。
- 左右内边距：20px。
- 悬停：向上移动 1px，亮度轻微提高。
- 按下：`scale(0.96)`。

**再走一次**

- 无填充。
- 文字使用结算页浅色文字。
- 下方 1px 同色相下划线，仅在悬停和键盘焦点时出现。
- 命中高度不低于 40px。

### 6.5 深度系统

项目默认不使用卡片和投影。空间层级由以下方式建立：

- 纸页与 letterbox：4% 以上的明度差。
- 月光：径向透明度变化，不添加发光描边。
- 月亮：通过墨边扩散和亮度变化体现实体。
- 海水：通过颜色渗透和运动层次体现深度。
- 结算按钮：允许使用一层极浅阴影 `0 2px 8px rgba(7, 74, 115, 0.16)`。

## 7. 关键页面与演出

### 7.1 开场书写

**目标**：3 秒内让玩家和旁观者确认“这是一首可以进入的诗”。

1. 纸张在 500ms 内淡入。
2. 24 个字符按 60ms 间隔写入。
3. 字符不模拟键盘光标，不使用打字机竖线。
4. 最后一字完成时，纸张整体下沉 2px 后回位，持续 180ms。
5. 方块从“床”字内部落下，落地只回弹一次，幅度 4px。
6. 右下角出现“按住左右两侧，开始走”。
7. 任意输入后提示立即淡出。

### 7.2 “光”

- 局部：光字墨色在 160ms 内扩散 1px 后恢复。
- 路径：80px 暖色光晕以字符为起点，随后跟随玩家。
- 页面：纸张从 `--paper` 向 `--paper-warm` 过渡。
- 总时长：2 秒。
- 光晕不使用纯白中心，不制造游戏技能特效感。

### 7.3 “霜”

- 局部：霜花从“霜”向玩家方块扩散，方块出现细裂纹。
- 路径：玩家立即停止，输入锁定 800ms。
- 页面：纸张整体向冷白偏移，幅度小于子弹时间。
- 解冻：霜花分裂为少量薄片向下飘落，300ms 内结束。
- 禁止高密度雪花粒子或蓝色冰冻滤镜。

### 7.4 卷轴换行

- 第一行向左移出并淡出。
- 第二行从下方上移到原第一行位置。
- 新玩家方块从左侧淡入，继承原水平速度。
- 总时长固定 200ms。
- 只动画 `transform` 和 `opacity`。
- 过渡期间锁输入，但保留按住状态。

### 7.5 子弹时间

- 触发点：“举”字中心。
- 世界速度降至 30%，持续 2 秒实时。
- 环境从暖白转冷灰。
- 第一行“明月”上下浮动 ±2px。
- 提示“试着……抬头看看？”在 200ms 后出现。
- 玩家仍然可以向左回退并重新触发，不产生一次性锁死。

### 7.6 撞月

这是首要传播画面，必须优先投入动效和真机调试时间。

| 时间 | 画面 |
|---:|---|
| 0 至 200ms | 方块向上冲出约 115px，拖尾拉长 |
| 200 至 400ms | 方块穿过第一行，整行裂成约 60 个墨色碎片 |
| 400ms | 时间恢复正常 |
| 400 至 700ms | 原“明月”位置生成 32px 墨色圆月并上升 8px |
| 700 至 1200ms | 方块落回第二行起点，后半句由半透明变实墨 |

动效规则：

- 字符碎片保持墨色，不使用彩色爆炸。
- 碎片速度由中心向外发散，但整体重心轻微向下，像铅字脱落。
- 第一行破碎后永久消失，留白成为“天空”。
- 月亮边缘不是几何硬圆，允许 1 至 2px 墨迹晕边。
- 方块落地后不做夸张弹跳，保持克制。

### 7.7 下沉与换页

1. 玩家在“低”字停稳并完成下滑保持。
2. 方块在 300ms 内下沉半格，“低”字向下弯曲。
3. 纸页从底部向上褪白，月亮被卷入并跟随下移。
4. 第 800ms 至 1000ms 进入短暂全白。
5. 第二关先显示月亮落定，600ms 后正文出现。

必须让月亮在换页过程中持续可追踪，不能先消失再在下一页重新生成。

### 7.8 第二关开场

**设计补充建议**

- 月亮落在画面右上正文之外，初始为墨色。
- 月亮落定后从中心透出暖金，600ms 内完成。
- 《故乡》正文按行淡入，不再逐字打字，避免重复第一关开场。
- 每行间隔 70ms，从包含“深蓝的天空中”的行开始，向上下两侧展开。
- 玩家最后落在第一行左侧基线，获得立即可控感。

### 7.9 月亮缝合

玩家经过“圆月”：

- “圆月”两字轻微向上位移 2px 后回落。
- 真月亮亮度提高，向上移动 6px。
- 月亮与“圆月”之间出现极淡的垂直墨痕联系，400ms 后消失。
- 提示“这一轮，刚才撞下来过”出现。
- 不再使用第二次爆裂，避免削弱第一次撞月。

### 7.10 海水变蓝

海水终章由“字符生成”“颜色变化”“页面材质变化”三条轨道同时推进。

**阶段 A，越界**

- 右侧纸张被水平拉开 200px。
- 第一个“海”在方块脚下生成。
- 镜头开始跟随，原文离开左侧。
- 月亮由暖金偏向银白，但保留暖色核心。

**阶段 B，无尽的海**

- 每前进一格生成一个“海”。
- 1 至 8 个保持墨黑。
- 9 至 24 个逐步混入 `--sea`。
- 少量“海”上下浮动，幅度不超过 6px。
- 空隙只出现 2 至 3 次，节奏间隔不得相等。

**阶段 C，页面蓝化**

- 约第 30 个“海”后，蓝色由笔画向纸张扩散。
- 背景依次经过 `--sea-mist`、`--sea`。
- 文章、作者和重置以“溶墨”方式消失。
- “海”字收束为横向波纹。
- 月亮倒影成为唯一垂直形态。
- 玩家锁输入并漂浮到画面中部。

**阶段 D，结算**

- 文案从海面下方上浮 12px，透明度渐入。
- 数据延后 300ms 出现。
- 两个按钮延后 600ms 出现。
- 8 秒无操作回到待机页。

## 8. 动效与声音规范

### 8.1 动效曲线

| 类型 | 时长 | 曲线 |
|---|---:|---|
| 按钮按压 | 100ms | `ease-out` |
| 图标状态切换 | 120ms | `cubic-bezier(0.2, 0, 0, 1)` |
| 普通位移 | 180 至 240ms | `cubic-bezier(0.16, 1, 0.3, 1)` |
| 提示淡入淡出 | 800ms | `ease-in-out` |
| 页面色温变化 | 500 至 800ms | `ease-in-out` |
| 关键舞台演出 | 按分镜固定 | GSAP timeline |

不使用弹簧、弹性或连续回弹曲线。开场方块落地允许一次受控回弹。

### 8.2 减少动态

开启 `prefers-reduced-motion` 时：

- 保留所有玩法时序和状态变化。
- 取消字符浮动、墨迹拖尾和页面震动。
- 撞月改为快速裂解加淡出，不使用高速飞散。
- 海字波动幅度降为 2px。
- 页面蓝化仍需保持至少 4 秒，避免因瞬时闪变造成不适。

### 8.3 声音层级

| 层级 | 声音 | 设计目的 |
|---|---|---|
| 环境 | 低音量纸张摩擦、后段海浪 | 建立空间，不抢夺文字 |
| 交互 | 轻触纸面、霜裂、撞墙 | 解释玩家动作 |
| 事件 | 撞月、下沉、入海 | 强化关键记忆 |

声音不能承载唯一反馈。关闭音频后，所有机关仍须可理解。

## 9. 输入与可访问性

### 9.1 输入映射

状态机只消费以下意图：

```text
hold-left
release-left
hold-right
release-right
jump
sink-attempt
cancel-sink
reset
```

键盘和触屏监听器分别生成意图，不相互模拟 DOM 事件。

### 9.2 触屏规则

- 左右半屏各占 50%，无中间死区。
- 所有触控区域启用 `touch-action: manipulation`。
- 下沉期间使用 `overscroll-behavior: contain`，避免浏览器页面滚动。
- 上滑和下滑只在单一触点下判定。
- 多指输入时保留第一个有效触点，其余忽略。

### 9.3 键盘规则

- `keydown` 的 `repeat` 不得重复触发 jump。
- 页面失焦时清空所有 hold 状态。
- 重新获得焦点后必须等待新输入，不能自动继续移动。
- 重置按钮可通过 Tab 聚焦和 Enter/Space 激活。

### 9.4 可访问性底线

- 重置与结算按钮必须有可见焦点态。
- 所有 icon-only 控件必须有 `aria-label`。
- Canvas 只承担视觉效果，关键信息必须保留在 DOM。
- 结算用时和步数使用可读文本，不能只绘制进 Canvas。
- 不依赖颜色区分可交互状态。
- 关键震动和闪动频率不得接近高风险频闪区间。

## 10. 响应式与设备表现

本项目不做传统响应式重排，只做固定画布等比缩放。

| 环境 | 行为 |
|---|---|
| 412 × 732 及以上竖屏 | 1:1 或按高度缩放，画布居中 |
| 窄于 412px 的竖屏 | 整体缩小，字符相对坐标不变 |
| 桌面横屏 | 竖屏画布居中，两侧 letterbox |
| 超宽屏 | 不放大超过可读上限，避免字符和粒子失真 |
| 横屏手机 | 提示旋转设备，或继续居中显示缩小后的竖屏画布 |

真机验证尺寸至少包含：

- 375 × 667
- 390 × 844
- 412 × 732
- 1440 × 900 桌面浏览器

## 11. 设计守则

### 11.1 必须做

1. 保持首屏像真实阅读页。
2. 所有字符都基于同一固定网格。
3. 每个机关同时提供局部、路径、页面三层反馈。
4. 撞月、月亮跟随和海水变蓝必须优先保证质量。
5. 提示保持文学语气，并在玩家成功后立即消失。
6. 所有关键演出需在两米外可辨认。
7. 颜色变化必须服务叙事顺序。
8. 桌面和移动端使用同一内部坐标系。

### 11.2 禁止做

1. 不添加卡片、血条、关卡进度条、虚拟摇杆或常驻操作说明。
2. 不提前高亮机关字。
3. 不使用紫蓝渐变、玻璃拟态或霓虹光效。
4. 不使用圆角胶囊作为默认控件。
5. 不把月亮画成带纹理的写实天体。
6. 不让粒子特效遮住正文和玩家方块。
7. 不用自动换行或设备差异改变机关坐标。
8. 不用声音替代视觉反馈。
9. 不在撞月之外重复使用大规模文字爆裂。
10. 不把结算页设计成常见手游胜利弹窗。

## 12. 实现策略

### 12.1 CSS 策略

**Vanilla CSS + CSS Custom Properties 单一策略。**

不引入 Tailwind、CSS Modules 或 CSS-in-JS。GSAP 只控制 timeline、transform、opacity 和必要的 CSS 变量。Canvas 只绘制粒子、光晕和颜色渗透。

### 12.2 图层职责

| 技术 | 负责 | 不负责 |
|---|---|---|
| DOM | 正文字符、作者、提示、按钮、可访问文本 | 大规模粒子 |
| GSAP | 字符移动、整行过渡、舞台时序 | 状态机 |
| Canvas | 墨迹、霜花、碎片、海水扩散 | 可点击控件和正文 |
| JavaScript 状态机 | 输入、机关、关卡状态、检查点 | 视觉样式 |
| JSON | 文本、坐标、机关配置 | 临时动画状态 |

### 12.3 性能预算

- 首帧渲染：二维码进入后 3 秒内完成。
- 动画目标：现场设备稳定 60fps，最低不低于 45fps。
- 同屏 Canvas 粒子建议上限：撞月 80，霜花 30，海水扩散按纹理块而非逐像素粒子。
- 总包体目标：5MB，硬上限 8MB。
- 图片资源原则上为 0。
- 音频单条不超过 50KB。

## 13. 待确认与冲突处理

### 13.1 第一关跳跃高度

PRD 6.5 的撞月演出要求方块上升约 115px，即 2.5 行高；10.3 又写“跳跃高度固定 = 1.5 行高”。

**建议结论**：

- 普通跳跃：1.5 行高。
- 第一关 BULLET_TIME 中的撞月：使用固定舞台演出 2.5 行高。
- 第一关定稿规则优先于通用技术建议。

### 13.2 第二关“撞碎”验收冲突

PRD 12 写“第二关至少包含一次跳跃撞碎”，但 7.5 和 7.6 只要求在“天空”处跳起并触碰月亮，没有可撞碎对象。

**建议结论**：将验收文字改为“第二关至少包含一次回读、一次跳跃触月、一次抵达目标句”。不要在第二关重复撞碎演出。

### 13.3 旧关卡 JSON 示例

PRD 10.2 的示例仍使用 4 行 × 5 字排版，与第一关最终 2 行 × 12 格冲突。

**处理结论**：实现时忽略该示例的行列数据，只参考 JSON 字段结构。第一关坐标以 6.2 和 6.3 为唯一真源。

### 13.4 第二关路径

第二关仍为初稿，机关分布、精确字符坐标和自动换行方向尚未锁定。

开发前至少需要补齐：

- 11 行文本的精确 `(row, col)` 坐标表。
- 每一行进入和离开的方向。
- “天空”阻挡后的落点。
- “带”字下沉到第 11 行的精确目标列。
- “刺去”强推开始和结束坐标。
- 海字空隙的位置或生成规则。

**开场页**

> 为第一关实现 1.2 秒撞月 timeline。背景 `#F0EFE9`，玩家 15px 方块在 200ms 内上升约 115px，拖尾持续 350ms。第 200 至 400ms 将第一行 12 个宋体字符裂解为约 60 个 `#20201E` 墨色碎片并永久移除。第 400 至 700ms 在原“明月”位置生成 32px 墨色圆月并上升 8px。第 700 至 1200ms 玩家落回第二行起点，后半句 opacity 从 0.35 到 1。只动画 transform 和 opacity。

**海水终章**

> 在固定画布内实现 18 秒海水终章。前 8 个“海”保持 `#20201E`，第 9 至 24 个逐步过渡到 `#176B9B`，少量字符上下浮动不超过 6px。约第 30 个“海”后，颜色从字的笔画向背景扩散，纸张依次变为 `#9EC4D6` 和 `#176B9B`。作者与正文溶解，海字收束为横向波纹，暖金月亮保留并产生垂直倒影，玩家变为 `#074A73`。蓝化过程至少持续 12 秒。

**结算页**

> 在 `#176B9B` 海面上设计无弹窗结算。主句“你真的游到了海水变蓝。”使用 Source Han Serif SC、24px、500 字重、36px 行高。下方显示用时和步数，16px，启用 tabular numbers。主按钮背景 `#E8D9A8`、文字 `#074A73`、高度 44px、圆角 4px。次按钮无填充，仅在 hover 和 focus 显示下划线。保留画面右上月亮和倒影，不添加卡片或胜利图标。
